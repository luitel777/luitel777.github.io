Producer Consumer Problem

Aagaman Luitel | words: 906 | 4 minutes | 2024.02.29
I was given a presentation task on this topic and I found it super interesting. Prior to this I had never properly used multithreading in my programs so discovering this was super fun and insightful. Producer Consumer Problem (Bounded buffer porblem)The concept is simple. There are two working entity (functions/threads) and one buffer. producer -> produces data consumer -> consumed data buffer -> Common shared buffer/data One simple example will be, suppose I have a stock analysis program, producer will generate stock values, profit and loss and so on and put theat generated value in the buffer, consumer takes that value and generates a real time graph.
